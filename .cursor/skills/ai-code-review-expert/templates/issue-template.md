# 问题记录模板

> 用于统一记录代码审查中发现的问题

## 📝 标准格式

````markdown
### 问题 #N [严重程度标识] | `文件路径:行号` | ⏳待修复

**维度**：[所属检查维度]

**问题代码**：
```[语言]
[引用原始代码片段，使用注释标注问题点]
```

**问题说明**：
- [描述具体问题]
- [说明可能的风险]
- [分析触发条件]

**修复建议**：
```[语言]
[给出具体的修复代码]
```

**理由**：[说明为什么要这样修改，有什么好处]

**参考资源**：[可选] [相关文档链接]
```
````

---

## 🎨 严重程度标识

使用以下标识符表示问题严重程度：

- `🔴严重` - 必须立即修复
- `🟠中等` - 建议尽快修复
- `🟡轻微` - 可选优化
