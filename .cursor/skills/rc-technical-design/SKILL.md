# 技术方案设计

基于需求分析文档中识别的改动点，制定详细技术方案。

## 文档结构

按以下顺序组织：

1. **整体流程图（如果有，必须放最上方）** — 使用 mermaid 时序图展示完整业务流程
2. **每个改动点的详细设计** — 现状逻辑 + 改动方案 + 流程图
3. **补充说明（可选）** — 如配置示例、关键设计说明

## 每个改动点设计要求

1. **文字说明现状逻辑**（使用步骤模版格式，见下方）
2. **文字描述改动方案**
3. **绘制流程图辅助理解**（使用 mermaid，禁止 ASCII 绘图）

### 步骤模版格式

```
第一步：[操作名称]
1. [具体步骤描述]
2. [具体步骤描述]

第二步：[操作名称]
1. [具体步骤描述]
...
```

完整示例参见 [💡 示例](./examples.md)

## 禁止内容

单元测试、测试验证、依赖关系、回滚方案、后续优化、技术风险、观测、应急、上线计划、工作量评估

详见 [🚫 禁止事项](./reference/prohibited-items.md)

## 输出要求

将完整方案保存到 `AICodingDoc/xxx需求/XXX需求技术方案.md`

文档格式详见 [📐 技术方案文档格式](./reference/design-document.md)
