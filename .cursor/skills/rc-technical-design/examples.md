# 开发工作流示例

## 现状逻辑步骤描述示例

以下是技术方案设计阶段中，描述现状逻辑的实际填写示例（来自实际业务场景：订单冲突检测）：

```
第一步：参数验证和初始化
1. 校验输入参数：自动抢单任务列表、已接收订单、弹窗配置不为空
2. 初始化冲突任务列表 receiveConflictForReceiveTaskList
3. 初始化冲突时间映射表 conflictTimeMap

第二步：提取订单时间信息
1. 检查司机预期出发时间是否为空，为空则退出
2. 获取司机预期出发时间 driverExpectDepartureTime
3. 计算司机预期到达时间 driverExpectArrivalTime = 出发时间 + 订单时长

第三步：获取冲突配置
1. 根据是否同城获取对应的冲突时间配置 conflictTimeConfig
2. 配置为空则退出方法
3. 基于配置扩展时间范围：出发时间提前，到达时间延后

第四步：过滤抢单任务
1. 遍历所有自动抢单任务
2. 跳过空对象
3. 排除当前订单对应的抢单任务

第五步：绑定司机订单信息
1. 识别沿途拼和临时行程类型的抢单任务
2. 从数据库查询并绑定对应的司机订单信息
3. 为时间计算和页面渲染提供数据支持

第六步：获取抢单时间区间
1. 调用 getAcceptTime 方法获取抢单任务的有效时间区间
2. 传入实时开始时间配置作为参数
3. 时间区间为空则跳过该任务

第七步：冲突检测
1. 遍历每个时间区间，验证格式正确性
2. 使用区间重叠算法：出发时间 < 区间结束时间 && 区间开始时间 < 到达时间
3. 检测到冲突则记录：任务GUID、开始时间、结束时间、重叠时长
4. 将冲突任务添加到列表并跳出循环

第八步：构建返回结果
1. 检查是否存在冲突任务
2. 将冲突任务转换为响应对象格式
3. 设置冲突信息到弹窗DTO：时间映射、配置、任务列表
4. 议价接单场景下设置专门的议价冲突弹窗配置
```
