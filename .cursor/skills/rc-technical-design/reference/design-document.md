# 技术方案文档格式

## 文档命名和存储

### 文档命名

文档名称：`XXX需求技术方案.md`（`XXX` 为需求简称）

### 存储位置

保存在：`AICodingDoc/xxx需求/` 目录下

## 文档内容要求

技术方案设计阶段，结合需求分析的结果，制定详细的技术方案。

### 文档结构

**按以下顺序组织**：

1. **整体流程图（如果有，必须放在最上方）**
   - 使用 mermaid 时序图展示完整业务流程
   - 让读者先理解全局，再看具体改动点

2. **改动点详细设计（必须）**
   - 针对需求分析阶段识别的每个改动点进行详细设计
   - 每个改动点包含：现状逻辑、改动方案、流程图

3. **补充说明（可选）**
   - 如配置示例、关键设计说明等

### 基本要求

- 此阶段应该针对分析阶段所有改动点进行详细设计
- 方案设计涉及到的绘图使用 mermaid 绘图
- 每一个改动点的详细设计，都应该先用文字说明具体的现状逻辑，然后写出具体的改动点，再用绘图辅助理解，让整体的方案设计更直观，更专业
- 不要使用 ASCII 绘图

### 禁止包含的内容

**严格禁止**以下与详细设计无关的内容：
- ❌ 单元测试设计
- ❌ 测试验证计划
- ❌ 依赖关系说明
- ❌ 回滚方案
- ❌ 后续优化建议

## 现状逻辑描述格式

现状逻辑按照步骤模板格式编写，在方案文档中应该是一个代码块。

### 格式规范

```
第一步：[操作名称]
1. [具体步骤描述]
2. [具体步骤描述]
3. [具体步骤描述]

第二步：[操作名称]
1. [具体步骤描述]
2. [具体步骤描述]

第三步：[操作名称]
...
```

### 编写要求

- 每个步骤应包含：操作名称、具体的执行步骤、关键数据和逻辑判断
- 步骤之间保持空行
- 使用代码块（三个反引号）包裹

完整示例参见 [examples.md](../examples.md)
