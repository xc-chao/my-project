# 需求分析

在开始前，展示需求输入模版，要求用户提供需求信息。

完整模版见：[📋 需求输入模版](./reference/requirements-template.md)

**强制要求**：不做工作量预估

## 第一步：需求理解与信息收集

1. 阅读产品PRD（如提供语雀链接，通过 yuque mcp 获取内容）
2. 识别需求的核心目标和业务价值
3. 明确各技术栈的分工和协作关系
4. 搜索 codebase，定位相关的现有代码和模块

## 第二步：改动点识别与分析

1. 系统化分析当前技术栈需要做的所有改动
2. 识别涉及的现有模块、类和方法

## 输出要求

将分析结论保存到 `AICodingDoc/xxx需求/XXX需求分析.md`

对每个改动点需明确：
- 改动位置（类名、方法名、文件路径）
- 改动类型（新增/修改/删除）
- 改动原因和目的（简短说明）
- 核心逻辑描述

文档格式和改动点聚合原则详见 [📄 需求分析文档格式](./reference/analysis-document.md)

## 需求澄清

分析完成后，至少提出 3 个问题，确保所有疑问都澄清。
