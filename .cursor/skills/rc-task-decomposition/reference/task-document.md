# 任务拆解文档格式

## 文档命名和存储

### 文档命名

文档名称：`XXX需求任务拆解.md`（`XXX` 为需求简称）

### 存储位置

保存在：`AICodingDoc/xxx需求/` 目录下

## 文档内容要求

详细阅读技术方案文档，将方案拆解为具体可执行的开发任务列表。

## 格式要求

使用 checklist 格式：
- 使用 `- [ ]` 标记未完成任务
- 使用 `- [x]` 标记已完成任务
- 不包含时间估算
- 按模块或功能分组

### 示例格式

```markdown
# XXX需求任务拆解

## 模块A
- [ ] 任务1描述
- [ ] 任务2描述

## 模块B
- [ ] 任务1描述
- [ ] 任务2描述
```

## 维护要求

在后续实施过程中及时更新当前实施进展，勾选已完成的任务。

### 更新方式

- 完成任务后，将 `- [ ]` 改为 `- [x]`
- 可以在任务后添加备注（如"当前正在进行"）
- 及时同步最新进展
