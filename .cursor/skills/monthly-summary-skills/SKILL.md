# 月度总结技能

用于基于 Git 提交生成精简版 markdown 工作总结，并在确认后上传到语雀。

## 快速开始

1. 先读取 [Monthly-Summary.md](Monthly-Summary.md)。
2. 从用户输入中提取时间范围和补充说明。
3. 先执行 `git config user.name` 获取作者名。
4. 再按时间范围获取提交、统计文件类型、分析重点提交和 diff。
5. 只保留 `3-4` 个主要工作点，按模块输出 markdown。
6. 初稿后用 `AskQuestion` 给出：
   - `很满意，接受保留`
   - `继续精简`
7. 最终稿确认后，再用 `AskQuestion` 询问是否上传到语雀。

## 必须遵守

- 不要只看 `git log --oneline` 就直接总结，必须结合文件列表和代表性 diff。
- 不要把零碎小改动逐条罗列进总结。
- 总结必须分模块归类，默认包含：
  - 总体概况
  - 功能开发
  - Bug 修复
  - 技术优化
- 输出必须是精简版 `markdown`。
- 涉及新页面或新模块时，要预留图片占位。
- 上传语雀前，必须先读取对应 Yuque MCP 工具描述文件。

## 输出目标

- 让读者快速看懂这段时间做了什么
- 聚焦结果、模块、价值，而不是 commit 流水账
- 默认保留 `3-4` 个主要工作点
