 • 选题的背景与意义
选题背景：随着移动互联网与即时通信技术的深度融合，微信小程序凭借“即用即走、无需安装”的轻量化特性，已成为中小型电商服务的重要载体。然而，当前多数购物类小程序仍存在功能同质化严重、交互体验薄弱、缺乏智能化服务能力等问题：一方面，商品浏览与下单流程缺乏个性化引导，难以满足用户差异化需求；另一方面，购前咨询与售后服务高度依赖人工客服，不仅响应滞后、人力成本高，还易造成用户体验断层。与此同时，大模型技术的快速演进为嵌入式智能对话系统提供了低成本、高可用的技术路径。在此背景下，设计并实现一个融合 AI 能力、覆盖完整交易闭环的智能购物微信小程序，具有迫切的现实需求和明确的应用价值。
本项目基于微信小程序前端框架，结合 Node.js 与 Express 构建后端服务，采用 MySQL 作为核心数据库，开发一套功能完备的智能电商系统。系统涵盖用户注册登录、商品分类展示与搜索、购物车管理、订单生成及全生命周期状态追踪等标准电商功能，并创新性集成 DeepSeek、OpenAI 等第三方大模型 API，构建内嵌式智能客服聊天模块。该模块支持用户以自然语言就商品详情、使用方法、物流进度等进行实时咨询，系统可结合当前商品上下文生成精准、语义连贯的回复，从而打造“浏览—咨询—下单—售后”一体化的智能移动端购物体验。

选题意义：
 • 技术探讨价值：本系统深入探索微信小程序与 Node.js 服务端在高并发、低延迟场景下的高效协同机制，验证 MySQL 在电商类应用中对读写密集型操作的稳定性与可扩展性，为轻量级全栈开发提供实践参考。
 • AI融合创新：通过将大模型能力轻量化集成至移动端客服场景，突破传统规则引擎或人工响应的局限，在不显著增加系统复杂度的前提下，实现语义理解与智能应答，推动 AI 技术在小微商业场景中的落地应用。
 • 工程范式贡献：项目形成“小程序前端 + Node.js 后端 + 大模型智能服务”三位一体的技术架构，为同类轻量级智能应用（如校园商城、社区团购、个体店铺）提供可复用、易扩展的开发范式与部署方案。
 • 社会应用价值：面向个体商户、校园创业团队等资源有限的用户群体，提供一套低成本、免安装、易维护且具备智能交互能力的电商解决方案，有效降低数字化经营门槛，助力小微经济数字化转型，具有良好的推广前景与社会价值。
2、课题研究的基本内容与拟解决的主要问题
基本内容：
 • 用户管理模块
 • 支持微信授权一键登录，同时兼容手机号 + 验证码方式完成注册与登录；
 • 用户可完善个人信息，包括昵称、头像、收货地址管理；
 • 提供密码找回功能，通过绑定手机号接收验证码重置密码；
 • 实现登录态持久化与安全校验机制，保障用户会话安全。
 • 商品展示与管理模块
 • 商品详情页包含图文介绍、价格、库存状态、用户评价等关键信息；；
 • 支持商品按分类展示、关键词搜索，并提供热销商品、新品推荐等导购功能；
 • 后台管理员可添加、编辑、上下架商品，支持图片上传与富文本描述；
 • 用户可查看他人对商品的评价内容，辅助购买决策。
 • 购物车与订单模块
 • 用户可将商品加入或移出购物车，并修改商品数量；
 • 支持多商品合并下单，生成唯一订单编号；
 • 订单状态全生命周期管理，包括：待付款、待发货、已发货、已完成、已取消；
 • 用户可查看历史订单记录，并提交售后申请（如退货、换货）。
 • 智能客服聊天模块（AI集成）
 • 小程序内嵌聊天窗口，仅限已登录用户使用；
 • 用户提问时，前端自动将当前商品 ID、名称、描述等信息作为上下文传递至后端；
 • 支持通用问题（如“怎么退货？”）与商品专属问题（如“这款手机支持快充吗？”）；
 • 聊天记录本地缓存，支持历史会话查看。
 • 后台管理模块
 • 管理员账号预置，可登录小程序管理后台；
 • 功能包括：商品管理、订单审核与状态更新、用户反馈查看；
 • 数据统计概览（日订单量、热销商品等）。
 • 数据存储与接口
 • 基于 MySQL 设计用户、商品、订单、聊天记录等表结构；
 • 使用 Express 构建 RESTful API，供小程序前端调用；
 • 实现数据增删改查、事务处理（如下单时扣减库存）、分页查询等操作。
 • 安全性：用户密码加盐哈希存储；敏感接口（如支付、订单修改）需身份鉴权；AI 接口密钥严禁暴露前端，仅由后端调用。
 • 兼容性：适配主流安卓/iOS 微信客户端，UI 使用 Vant Weapp 组件库保证一致性；
 • 易用性：界面布局清晰，AI 功能入口直观，无技术背景的用户可在 5 分钟内掌握核心操作。
拟解决的主要问题：
 • 功能碎片化与体验割裂问题：当前多数小型购物小程序功能单一，缺乏从商品浏览、智能咨询到下单售后的完整闭环。本系统通过整合商品展示、AI 客服、购物车、订单管理等模块，构建一体化购物流程，提升用户操作连贯性与满意度。
 • AI 功能与框架融合问题：实现 DeepSeek、Coze 等 AI 大模型接口与 Next.js 框架的稳定集成，解决接口调用超时、失败处理等异常情况，保障 AI 功能的可用性。
 • 多兼容与交互一致性问题：不同品牌手机及微信版本对小程序渲染存在差异。系统采用 Vant Weapp 组件库 + 响应式布局，确保在主流 Android 与 iOS 设备上界面统一、操作流畅。
 • 高并发下数据一致性问题：在用户同时下单、库存扣减等场景中，易出现超卖或数据不一致。系统通过 MySQL 事务机制与乐观锁策略，保障订单生成与库存更新的原子性与准确性。
 • 用户权限与安全防护薄弱问题：部分小程序未对敏感接口做鉴权，存在数据泄露或越权操作风险。本系统采用 JWT + Session 双重认证机制，对关键接口（如修改订单、访问聊天记录）实施严格权限校验，并对密码、AI 密钥等敏感信息加密或后端隔离处理。
 • AI接口调用稳定性问题：大模型 API 存在延迟、限流或返回异常的可能。系统在后端封装统一调用层，加入超时控制、错误重试、降级提示等机制，确保 AI 客服功能在异常情况下仍具备基础可用性。
3、课题研究方法、设计方案或论文撰写提纲
设计方案：
 • 系统架构设计：采用前后端分离架构。前端基于微信小程序原生框架（WXML/WXSS/JavaScript）开发；后端使用 Node.js + Express 构建 RESTful API 服务；数据库采用 MySQL 存储用户、商品、订单、聊天记录等数据；AI 智能客服通过后端调用 DeepSeek/OpenAI 接口实现。整体分为：小程序前端层、Node.js 服务层、MySQL 数据层、AI 能力层。
 • 数据流程设计：明确核心业务数据流：用户登录态通过 code 换取 session_key 并生成 token；商品浏览数据从 MySQL 查询返回；下单时开启事务，同步扣减库存并生成订单；用户提问时，前端携带商品 ID，后端拼接上下文调用 AI 接口并缓存问答记录，确保数据流转安全、完整、可追溯。
 • 界面与交互设计：遵循微信小程序设计规范，采用 Vant Weapp 组件库保证 UI 一致性。首页突出商品分类与推荐，商品详情页集成“AI 咨询”按钮，聊天窗口支持消息气泡与历史回溯，整体操作路径简洁，新用户可在 5 分钟内掌握核心功能。
 • AI 接口密钥仅保存于后端环境变量，前端无法访问；后端封装统一代理接口，接收用户问题与商品上下文，调用大模型 API 并处理超时（>8s）、返回格式错误、token 超限等情况，必要时返回友好提示或默认答案，保障服务鲁棒性。
 • 数据库设计：基于 MySQL 设计核心表结构、商品表、订单表、问答记录表等，定义字段类型与关联关系，优化查询语句，提升数据读写效率。
