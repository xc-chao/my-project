# 📖 ScanShop 项目运行指南

## 🎯 项目概述

ScanShop 是一个完整的微信小程序扫码购物系统，包含：
- ✅ 30个 REST API 接口
- ✅ 6张数据库表，50个商品
- ✅ 前后端完整实现
- ✅ 用户认证、购物车、订单、支付功能

## 🔄 完整的接口执行流程

### 示例：获取商品列表

#### 1️⃣ 用户操作
```
用户打开小程序 → 首页加载
```

#### 2️⃣ 前端发起请求

**文件路径**：`frontend/pages/index/index.js`
```javascript
// 第 28 行：页面加载
onLoad() {
  this.fetchProducts();  // 调用获取商品
}

// 第 72 行：获取商品列表
async fetchProducts() {
  const res = await getProducts(1, 20);  // 调用API封装
}
```

#### 3️⃣ API 封装层

**文件路径**：`frontend/utils/api.js`
```javascript
// 第 234 行：getProducts 函数
export const getProducts = (page, pageSize, category) => {
  let url = `/products?page=${page}&pageSize=${pageSize}`;
  return request(url);  // 调用网络请求
};
```

#### 4️⃣ 网络请求层

**文件路径**：`frontend/utils/request.js`
```javascript
// 第 11 行：BASE_URL
const BASE_URL = 'http://localhost:3000/api';

// 第 25 行：发起HTTP请求
wx.request({
  url: `${BASE_URL}${url}`,  
  // 完整URL: http://localhost:3000/api/products?page=1&pageSize=20
  method: 'GET',
  header: { 'Authorization': 'Bearer <token>' }
});
```

#### 5️⃣ 后端接收请求

**文件路径**：`backend/src/app.js`
```javascript
// 第 30 行：挂载路由
app.use('/api/products', productRoutes);
// 将 /api/products/* 的请求交给 productRoutes 处理
```

#### 6️⃣ 路由匹配

**文件路径**：`backend/src/routes/productRoutes.js`
```javascript
// 第 15 行：路由定义
router.get('/', asyncHandler(productController.getProducts));
// GET /api/products/ → 执行 productController.getProducts
```

#### 7️⃣ 控制器处理

**文件路径**：`backend/src/controllers/productController.js`
```javascript
// 第 23 行：控制器方法
exports.getProducts = async (req, res) => {
  // 获取查询参数
  const { page, pageSize, category } = req.query;
  // req.query = { page: '1', pageSize: '20' }
  
  // 构建查询条件
  const where = { status: 1 };
  if (category) {
    where.category = category;
  }
  
  // 调用 Sequelize 查询
  const { count, rows } = await Product.findAndCountAll({
    where,
    limit: 20,
    offset: 0
  });
  
  // 返回响应
  return success(res, { products: rows, pagination: {...} });
};
```

#### 8️⃣ Sequelize 查询数据库

**文件路径**：`backend/src/models/Product.js`

Sequelize 生成 SQL：
```sql
SELECT COUNT(*) FROM products WHERE status = 1;
SELECT * FROM products WHERE status = 1 LIMIT 20 OFFSET 0;
```

#### 9️⃣ MySQL 执行查询

**数据库**：`scanshop_db.products`

MySQL 服务器：
1. 接收 SQL 语句
2. 使用索引查找数据
3. 返回 20 条商品记录

**HTTP 响应**：

```json
  "message": "操作成功",
  "data": {

```

## 📊 数据流向

### 用户购物完整流程

```
1. 用户注册/登录
   POST /api/auth/login
   ↓ 返回 token
   
2. 浏览商品
   GET /api/products?category=饮料
   ↓ 返回 10 个饮料商品
   
3. 加入购物车
   POST /api/cart { productId: 1, quantity: 2 }
   ↓ 检查库存 → 创建/更新购物车记录
   
4. 查看购物车
   GET /api/cart
   ↓ 返回购物车列表 + 总价
   
5. 创建订单
   POST /api/orders
   ↓ 开启事务
   ├─ 验证库存
   ├─ 创建订单主表
   ├─ 创建订单明细
   ├─ 扣减库存（锁定）
   └─ 清空购物车
   ↓ 提交事务
   
6. 支付订单
   POST /api/payments/create
   ├─ 微信支付 → 调用微信API → 返回支付参数
   └─ 余额支付 → 扣除余额 → 立即完成
   
7. 支付成功
   POST /api/payments/callback（微信回调）
   ↓ 更新订单状态 → 更新支付记录
   
8. 查看订单
   GET /api/orders
   ↓ 返回用户的所有订单
```

## 🔐 核心技术实现

### 1. 用户认证（JWT）

```
用户登录 → 后端验证密码 → 生成 JWT token
前端保存 token → 后续请求携带 token
后端 authMiddleware 验证 token → 解析 userId
```

### 2. 购物车管理（MySQL存储）

```
添加商品 → 检查是否已存在
  ├─ 存在 → 数量 += quantity
  └─ 不存在 → 创建新记录
  
数据库：UNIQUE(user_id, product_id) 保证不重复
```

### 3. 库存管理（数据库事务）

```
创建订单：
  BEGIN TRANSACTION
    ├─ 验证库存
    ├─ 扣减库存（锁定）
    ├─ 创建订单
    └─ COMMIT（成功）或 ROLLBACK（失败）
    
取消订单：
  BEGIN TRANSACTION
    ├─ 增加库存（回滚）
    ├─ 更新订单状态
    └─ COMMIT
```

### 4. 支付集成

```
微信支付：
  创建支付 → 调用微信统一下单 API
  → 返回支付参数 → 前端调起微信支付
  → 用户支付 → 微信回调通知 → 更新订单状态
  
余额支付：
  创建支付 → 验证余额 → 扣除余额
  → 更新订单状态 → 立即返回成功
```

## 🎯 核心功能说明

### 1. 认证系统

**支持两种登录**：
- 微信小程序登录（wx.login + openid）
- 手机号密码登录（bcrypt 加密）

**JWT Token**：
- 有效期 7 天
- 携带在 Header: `Authorization: Bearer <token>`

### 2. 购物车系统

**数据存储**：
- MySQL 数据库（cart_items 表）
- 跨设备同步

**核心功能**：
- 智能数量合并（重复添加累加）
- 实时库存验证
- 选中状态管理
- 自动计算总价

### 3. 订单系统

**订单状态机**：
```
待支付(0) → 已支付(1) → 已完成(3)
   ↓
已取消(2)
```

**库存管理**：
- 创建订单：锁定库存（扣减）
- 取消订单：回滚库存（增加）
- 使用数据库事务保证一致性

### 4. 支付系统

**支付方式**：
- 微信支付（异步，回调通知）
- 余额支付（同步，立即完成）

**安全措施**：
- 签名验证
- 幂等性处理
- 金额校验

## ⚠️ 常见问题

### Q: 后端启动失败？
A: 检查 MySQL 是否运行，密码是否正确

### Q: 前端无法连接后端？
A: 确保服务器在运行（`lsof -i:3000`），勾选"不校验合法域名"

### Q: Vant 组件不显示？
A: 微信开发者工具 → 工具 → **构建 npm**

### Q: 中文乱码？
A: 导入数据时加 `--default-character-set=utf8mb4`

### Q: 图片不显示？
A: 图床可能有防盗链，使用占位图片服务
