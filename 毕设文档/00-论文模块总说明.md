# 论文模块总说明

本文档目录用于存放《基于 Node.js 与 JavaScript 实现购物小程序的设计与实现》毕业论文的模块化正文内容。为了便于后续整理成约 40 页 Word 论文，本文将系统内容按照业务模块和技术模块拆分为多个独立文件，每个文件均可作为论文正文中的一节或若干小节直接合并使用。

本系统围绕“商品浏览、搜索筛选、商品详情、购物车、订单确认、历史订单、售后申请、AI 智能咨询、后台管理”的完整购物流程展开。前端部分基于 `uni-app` 与 Vue 语法组织页面，主要页面集中在 `frontend/miniprogram/src/pages` 目录；服务请求层集中在 `frontend/miniprogram/src/services` 目录；用户与购物车状态分别由 Pinia store 管理。后端部分基于 `Node.js + Express` 实现，入口文件为 `backend/src/app.js`，并按照 `auth`、`product`、`cart`、`order`、`chat`、`address`、`afterSale`、`feedback`、`admin` 等业务域划分模块。各模块通常采用 `routes -> controller -> service -> repository` 的分层结构，体现了较清晰的接口边界和业务职责。

后续 Word 排版时，建议按以下顺序合并：

1. `01-系统总体架构与技术选型.md`
2. `02-用户认证与个人中心模块.md`
3. `03-首页商品展示与搜索模块.md`
4. `04-商品详情与反馈模块.md`
5. `05-购物车模块.md`
6. `06-订单与地址模块.md`
7. `07-售后申请模块.md`
8. `08-AI智能客服模块.md`
9. `09-后台管理模块.md`
10. `10-系统安全与接口规范.md`
11. `11-系统测试与运行部署.md`

每个模块中均保留了“图表占位”，后续可以插入系统截图、流程图、E-R 图、接口表或测试表。若按照本科毕业论文常见排版格式（小四号字体、1.5 倍行距）整理，当前内容可作为 40 页论文的主体材料继续扩写和精修。
