# 首页商品展示与搜索模块

## 1 模块概述

首页商品展示与搜索模块是用户进入系统后的主要浏览入口。首页页面位于 `frontend/miniprogram/src/pages/home/index.vue`，搜索页面位于 `frontend/miniprogram/src/pages/search/index.vue`，商品接口封装在 `frontend/miniprogram/src/services/productService.ts`。后端商品模块位于 `backend/src/modules/product`，由 `product.routes.js`、`product.controller.js`、`product.service.js` 和 `product.repository.js` 组成。

该模块承担商品信息展示、分类切换、关键词搜索、排序筛选和商品详情跳转等职责，是购物流程的起点。用户可以从首页浏览推荐商品，也可以进入搜索页根据关键词查找目标商品。

## 2 首页功能设计

首页主要面向用户的快速浏览和分发。根据现有页面文档和代码结构，首页包括顶部品牌区域、搜索入口、Banner 视觉区、分类筛选、热销商品列表和 AI 导购入口。用户点击搜索框可进入搜索页面，点击分类可切换商品列表，点击商品卡片可进入商品详情页，点击 AI 悬浮入口可进入聊天页面。

首页的功能设计遵循“先吸引、再分发、再转化”的思路。顶部区域强化系统名称和搜索入口，Banner 区用于展示运营主题或活动内容，分类区域帮助用户缩小浏览范围，商品卡片则直接展示商品核心信息，如商品图片、名称、价格、标签和销量等。

## 3 搜索功能设计

搜索页用于满足用户主动查找商品的需求。用户输入关键词后，系统根据关键词请求后端商品列表接口，并返回符合条件的商品。搜索模块还支持筛选和排序，例如按分类、价格、热度或上新时间筛选商品。这样可以避免用户在商品数量较多时只能依赖首页推荐，提高商品查找效率。

前端 `productService.ts` 中通过 `buildProductQueryString` 方法构建查询参数，并提供 `getProductList`、`getProductDetail`、`searchProducts` 等方法。通过这种封装方式，页面不需要直接处理复杂查询字符串，而是通过对象参数表达搜索条件。

搜索页的状态通常包括：

- `keyword`：用户输入的关键词。
- `category`：当前分类条件。
- `sort`：当前排序方式。
- `products`：搜索结果列表。
- `loading`：接口加载状态。
- `empty`：结果为空状态。

## 4 后端商品查询实现

后端商品模块的核心在 `product.repository.js`。该文件提供商品查询、商品详情获取、分类归一化、条件构造和排序构造等能力。查询逻辑会根据前端传入的关键词、分类、排序等参数动态构建 SQL 条件。对于商品图集，后端会通过 `fetchGalleryMap` 获取商品图片并与商品主数据组装后返回前端。

商品查询接口通常包括以下能力：

1. 获取商品列表。
2. 按关键词搜索商品。
3. 按分类筛选商品。
4. 按价格、热度或时间排序。
5. 获取商品详情。
6. 获取商品分类。

该模块的后端实现强调字段结构统一。商品列表返回的数据既要满足首页商品卡片展示，也要满足搜索结果页展示；商品详情接口则需要返回更多信息，如图集、规格、营销标签、服务承诺和推荐商品等。

## 5 前后端交互流程

首页商品加载流程如下：

1. 用户打开首页。
2. 前端调用 `getProductList` 请求商品列表。
3. 后端商品路由接收请求并进入 controller。
4. service 层调用 repository 查询数据库。
5. repository 组装商品和图集数据。
6. 后端返回统一响应。
7. 前端将商品渲染为商品卡片。

搜索流程如下：

1. 用户进入搜索页并输入关键词。
2. 前端将关键词、分类和排序条件转换为 query 参数。
3. 后端根据 query 参数构造查询条件。
4. 数据库返回符合条件的商品记录。
5. 前端展示搜索结果或空状态。

## 6 模块实现特点

本模块的特点是将商品浏览入口和搜索能力统一到同一个商品服务体系中。前端页面可以根据不同场景调用同一组商品接口，后端也可以复用查询逻辑，从而避免首页、搜索页和详情页之间出现字段不一致的问题。

此外，系统通过商品卡片组件复用展示逻辑，使首页和搜索页的商品展示风格保持统一。后端采用动态条件构造方式，提高了搜索接口的灵活性。对于未来扩展而言，该模块可以继续加入销量排序、价格区间、品牌筛选、智能推荐等功能。

## 7 本章小结

首页商品展示与搜索模块是购物小程序的流量入口，也是用户完成商品发现的核心模块。该模块通过首页推荐、分类浏览和关键词搜索共同支撑商品查找需求，并通过统一商品接口和组件复用提高了系统维护性。
