# UniAppPencil 运行配置

## Windows 启动项目

默认启动 H5：

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\86157\Desktop\UniAppPencil\script\start.ps1
```

启动微信小程序模式：

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\86157\Desktop\UniAppPencil\script\start.ps1 mp-weixin
```

脚本会自动检查后端、前端、`node`、`npm`、`node_modules`，并检测后端健康检查地址：

```text
http://localhost:3001/health
```

如果后端已经在 `3001` 端口正常运行，脚本会跳过后端启动，只启动前端。

## VS Code / Cursor 任务

工作区文件 `UniAppPencil.code-workspace` 已配置两个任务：

- `Start UniAppPencil H5`
- `Start UniAppPencil Weixin`

在 VS Code / Cursor 中可以通过任务面板直接运行。

## Codex 免确认配置

当前 Codex 配置文件：

```text
C:\Users\86157\.codex\config.toml
```

已配置为默认全权限、无需确认：

```toml
sandbox_mode = "danger-full-access"
approval_policy = "never"

[windows]
sandbox = "elevated"

[projects.'c:\users\86157\desktop\uniapppencil']
trust_level = "trusted"
```

正常启动 Codex 即可读取上面的全局配置：

```powershell
codex -C C:\Users\86157\Desktop\UniAppPencil
```

也可以用下面命令显式指定：

```powershell
codex --ask-for-approval never --sandbox danger-full-access -C C:\Users\86157\Desktop\UniAppPencil
```

如果希望完全跳过确认和沙箱，用下面命令启动：

```powershell
codex --dangerously-bypass-approvals-and-sandbox -C C:\Users\86157\Desktop\UniAppPencil
```

最后一种方式会让 Codex 直接在本机执行命令，只适合你明确信任当前项目和当前会话时使用。
