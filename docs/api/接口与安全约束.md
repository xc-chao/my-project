# 接口与安全约束

## 设计原则

后端采用 `Express.js`，遵循模块化、控制器与业务分层、集中异常处理、统一鉴权、统一校验的方式组织接口。

重点约束来自两部分：

1. `express.mdc` 中的模块化、安全、错误处理、验证、中间件分层要求
2. `secure-claude-skills-rate-limiting` 中的限流建议，尤其是高风险与高成本接口保护

## 接口分组

### 公开接口

- `GET /api/products`
- `GET /api/products/:id`
- `GET /api/categories`
- `GET /api/reviews`

特点：

- 只读
- 可分页
- 可缓存
- 不暴露敏感字段

### 登录与认证接口

- `POST /api/auth/wechat-login`
- `POST /api/auth/sms-code`
- `POST /api/auth/sms-login`
- `POST /api/auth/logout`
- `GET /api/auth/profile`

约束：

- `sms-code`、`sms-login` 必须限流
- 登录成功后返回 `accessToken + session`
- 敏感日志脱敏

### 用户侧受保护接口

- `GET /api/user/addresses`
- `POST /api/user/addresses`
- `PUT /api/user/addresses/:id`
- `DELETE /api/user/addresses/:id`
- `GET /api/cart`
- `POST /api/cart/items`
- `PATCH /api/cart/items/:id`
- `DELETE /api/cart/items/:id`
- `POST /api/orders`
- `GET /api/orders`
- `GET /api/orders/:id`
- `POST /api/orders/:id/cancel`
- `POST /api/chat/sessions`
- `POST /api/chat/messages`
- `GET /api/chat/history`

约束：

- 全部要求登录
- 用户仅能访问自己的资源
- 订单创建和库存扣减必须走事务

### 管理后台接口

- `POST /api/admin/products`
- `PUT /api/admin/products/:id`
- `PATCH /api/admin/products/:id/status`
- `GET /api/admin/orders`
- `PATCH /api/admin/orders/:id/status`
- `GET /api/admin/stats/overview`

约束：

- 仅管理员角色可访问
- 商品上下架、订单状态变更要记录操作日志

## 限流策略

基于技能文档，默认采用按 IP 的窗口限流，对高风险接口使用更严格策略。

### 默认策略

- `5 req/min/IP`
- 超出后返回 `429 Too Many Requests`
- 返回 `Retry-After`、`X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`

### 必加限流的接口

- 登录接口
- 短信验证码接口
- AI 对话接口
- 搜索接口
- 图片上传接口
- 评论/反馈提交接口

### 特别说明

- `POST /api/chat/messages` 是高成本接口，必须同时具备登录校验和限流
- 生产环境建议将限流存储从内存切换到 `Redis`
- 限流触发需记录安全日志，便于识别暴力请求与恶意刷接口

## 校验与错误处理

### 参数校验

- 所有 `POST`、`PUT`、`PATCH` 请求统一走参数校验中间件
- 商品、订单、地址、聊天请求分别定义独立校验器
- 参数不合法时返回 `400`

### 统一错误结构

建议统一返回：

```json
{
  "success": false,
  "code": "RATE_LIMIT_EXCEEDED",
  "message": "请求过于频繁，请稍后再试",
  "requestId": "trace_xxx"
}
```

### AI 接口降级

AI 相关接口建议增加：

- 超时控制：`8s`
- 失败重试：`1~2 次`
- 降级提示：`当前智能客服繁忙，请稍后重试`
- 上下文裁剪：避免 token 过长

## 推荐中间件顺序

建议请求链路如下：

1. `requestId`
2. `helmet / security headers`
3. `cors`
4. `json parser`
5. `session`
6. `auth`
7. `rateLimit`
8. `validate`
9. `controller`
10. `errorHandler`

## 给设计稿的落地建议

在 `CreatPencil.pen` 的后端架构区，可以额外加一排“接口约束标签”：

- `JWT + Session`
- `Role Guard`
- `Rate Limit`
- `Validation`
- `Transaction`
- `AI Timeout/Fallback`

这样设计稿既能说明功能，也能说明工程可靠性。
