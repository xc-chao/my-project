# `CreatPencil.pen` 页面蓝图

## 目标

基于 `开题报告.md` 的功能范围、`drawio.png` 的系统结构图，以及后端 `Express.js` 分层和接口安全约束，为 `CreatPencil.pen` 规划一版适合答辩展示与后续实现对照的设计稿。

当前建议把 `.pen` 画布拆成 4 个主区域，从左到右排列，形成“用户端页面 -> 后端架构 -> 数据与 AI -> 项目目录”的完整叙事链路。

## 画布布局

### 1. 小程序用户端页面区

用于展示用户完整购物闭环，建议做成 2 行 4 列的手机稿矩阵。

建议页面如下：

1. 登录/授权页
2. 首页
3. 商品列表/搜索页
4. 商品详情页
5. AI 客服聊天页
6. 购物车页
7. 订单列表/订单详情页
8. 个人中心页

### 2. 后端架构区

保持和 `drawio.png` 一致的三层认知，但做得更“产品化”：

1. 接入层
2. 业务层
3. 基础设施层

建议卡片内容：

- 接入层：`Nginx / HTTPS`、`Express App`、`RESTful API`
- 鉴权层：`JWT + Session`、`RBAC 权限校验`
- 业务层：`Auth`、`User`、`Product`、`Cart`、`Order`、`Chat`
- 安全层：`参数校验`、`限流`、`异常处理`、`审计日志`
- 基础设施层：`MySQL`、`Redis(可选)`、`对象存储(商品图)`、`AI Provider`

### 3. 数据与 AI 区

建议使用表卡片 + 流程箭头的形式。

核心表：

- `users`
- `addresses`
- `products`
- `product_reviews`
- `cart_items`
- `orders`
- `order_items`
- `chat_sessions`
- `chat_messages`

AI 流程：

1. 小程序提交问题与商品上下文
2. 后端校验登录态、限流与参数
3. AI 适配层拼装 Prompt
4. 调用 `DeepSeek/OpenAI`
5. 落库聊天记录并返回回复

### 4. 项目目录区

在右侧放一张“项目文件树”卡片，展示前端、后端、文档的分层结构，和 `docs/architecture/目录结构方案.md` 保持一致。

## 每个页面的视觉重点

### 登录/授权页

- 顶部品牌标题：智能购物小程序
- 中部按钮：微信一键登录
- 次级入口：手机号验证码登录
- 底部说明：隐私政策、登录态持久化、安全校验

### 首页

- 顶部搜索框
- 轮播 Banner
- 分类宫格
- 热销商品区
- 新品推荐区
- 悬浮入口：`AI 购物助手`

### 商品列表/搜索页

- 顶部搜索与分类筛选
- 商品卡片流
- 排序方式：销量、价格、新品
- 空状态：暂无匹配商品

### 商品详情页

- 商品图轮播
- 标题、价格、库存、评价
- 规格选择
- 底部操作：加入购物车、立即购买
- 强化按钮：`问问 AI`

### AI 客服聊天页

- 顶部展示当前商品上下文
- 中部消息气泡流
- 底部输入框与发送按钮
- 侧边/顶部小提示：
  - 仅登录用户可用
  - 超时返回友好提示
  - 聊天记录本地缓存

### 购物车页

- 商品勾选
- 数量加减
- 合计金额
- 结算按钮

### 订单列表/详情页

- 状态 Tabs：待付款、待发货、已发货、已完成、已取消
- 订单卡片包含商品、金额、时间、状态
- 详情页展示物流、地址、售后入口

### 个人中心页

- 用户头像、昵称
- 收货地址管理
- 历史订单
- 客服记录入口
- 退出登录

## 后端约束可视化建议

后端区域不要只画技术名词，建议把约束画成状态标签或注释条：

- 认证：`JWT + Session`
- 校验：`express-validator / Joi`
- 限流：`登录、验证码、AI 对话接口默认 5 req/min/IP`
- 安全：`密码加盐哈希`、`AI Key 仅服务端存储`
- 稳定性：`AI 超时 8s`、`失败重试`、`降级文案`
- 数据一致性：`下单事务 + 库存扣减`
- 错误响应：`统一 JSON 错误结构`

## 接口标签建议

可以在接口箭头旁标 4 类标签：

- `Public`：商品列表、商品详情
- `Auth Required`：购物车、订单、地址、聊天记录
- `Admin Only`：商品管理、订单审核、统计概览
- `Rate Limited`：登录、验证码、AI 对话、搜索、文件上传

## `.pen` 落图顺序建议

当 Pencil 连接恢复后，建议按下面顺序绘制，避免画布混乱：

1. 先搭 4 个主分区容器
2. 再放 8 个手机页面骨架
3. 再补后端架构卡片
4. 然后补数据表和 AI 流程
5. 最后放项目目录树与图例说明

## 设计风格建议

- 整体风格：简洁、答辩友好、信息层级清晰
- 主色：微信蓝绿系或低饱和蓝色
- 卡片：圆角 + 轻阴影
- 字体层级：标题明显，正文克制
- 连接线：接口流向用蓝色，数据库流向用绿色，AI 调用用紫色
