# 后台信息管理页

## 页面目标

仅对管理员开放，用于集中查看商品管理、订单处理、用户反馈与数据看板，是后台运营入口页。

## 入口与出口

- 入口：`我的` 页面 `订单状态` 下方的 `后台信息管理` 卡片。
- 出口：商品管理子页、订单处理子页、反馈与看板子页。

## 子页面拆分

- `admin-products.md`：商品管理页。
- `admin-orders.md`：订单处理页。
- `admin-order-detail.md`：订单详情页。
- `admin-after-sales.md`：售后审批页。
- `admin-feedback.md`：反馈与看板页。

## 权限规则

- 只有 `admin` 身份登录后显示入口。
- 普通用户不展示入口。
- 即使手动输入路由，也需要前端拦截和后端接口鉴权双重保护。

## 主要区域

- 管理员概览头部。
- 数据指标卡片，如日订单、待处理售后、在售商品、待看反馈。
- 管理模块列表：商品管理、订单处理、用户反馈与数据看板。
- 热销商品排行。
- 最近反馈预览。
- Agent 调用监控。

## 交互说明

- 点击商品管理卡片，进入商品管理明细页。
- 点击订单处理卡片，进入订单处理或订单列表页。
- 点击反馈看板卡片，进入评价反馈与监控页。
- 页面首次加载时请求管理员概览接口。

## 字段/状态说明

- `overview`：管理员概览数据。
- `loading`：加载状态。
- `userStore.isAdmin`：管理员身份判断。

## 对应接口

- `GET /api/admin/overview`
- `GET /api/admin/products`
- `GET /api/admin/orders`
- `GET /api/admin/feedback`

## 对应 Pencil 页面

- `CreatPencil.pen` 后台信息管理页。
