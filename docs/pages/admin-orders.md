# 订单处理页

## 页面目标

供管理员统一查看订单流转状态，并推进支付确认、发货、签收完成等处理节点，同时进入订单详情与售后审批流。

## 入口与出口

- 入口：后台信息管理页中的 `订单处理` 卡片。
- 出口：后台信息管理页、售后相关页面。

## 权限规则

- 仅 `admin` 身份可访问。
- 普通用户不显示入口，接口层也必须校验管理员身份。

## 主要区域

- 订单处理说明头部。
- 汇总卡片：订单总数、待处理、配送中、已完成。
- 筛选栏：全部、待处理、配送中、已完成。
- 订单处理列表。
- 订单详情页与售后审批页入口。

## 交互说明

- 待支付订单可推进到待发货。
- 待发货订单可推进到已发货。
- 已发货订单可推进到已完成。
- 已完成或已取消订单仅展示，不再推进状态。
- 点击 `查看详情` 可进入订单详情页。
- 点击 `售后审批` 可进入售后审批页。
- 订单详情页中可补录物流单号并查看时间线。

## 字段/状态说明

- `orders`：订单列表。
- `currentFilter`：当前筛选项。
- `status`：订单流转状态。
- `orderId`：详情和售后聚焦使用的订单标识。

## 对应接口

- `GET /api/admin/orders`
- `GET /api/admin/orders/:id`
- `PATCH /api/admin/orders/:id/status`
- `PATCH /api/admin/orders/:id/logistics`
