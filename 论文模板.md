# 基于 Node.js 与 JavaScript 实现购物小程序的设计与实现

## 摘要

随着移动互联网和微信生态的持续发展，小程序因其即用即走、开发成本较低、使用门槛较小等特点，逐渐成为中小型电商系统的重要载体。针对传统购物小程序在商品展示、订单管理、用户交互和智能咨询方面存在的功能碎片化、体验割裂以及客服响应效率不足等问题，本文设计并实现了一套基于 Node.js 与 JavaScript 的智能购物小程序系统。系统采用前后端分离的开发模式，前端基于 uni-app 开发微信小程序界面，后端基于 Express 构建 RESTful API 服务，数据库采用 MySQL 进行数据持久化存储，并结合第三方大模型能力构建 AI 智能客服模块。

系统围绕“浏览商品、智能咨询、加入购物车、下单支付、订单追踪、售后处理”的完整业务闭环展开设计，重点实现了用户登录与信息管理、商品分类展示与搜索、商品详情浏览、购物车管理、订单流转、AI 聊天咨询以及后台管理等功能。与传统电商小程序相比，本系统在商品详情页中引入上下文感知的智能问答能力，使用户可以围绕商品属性、使用方式、物流与售后等问题进行自然语言咨询，从而提升了交互效率与用户体验。

在系统实现过程中，本文重点讨论了前后端接口设计、数据库表结构设计、事务处理机制以及 AI 接口调用的稳定性控制方案。测试结果表明，系统能够较好地满足基本购物场景的功能需求，具备一定的可扩展性和实际应用价值。

**关键词：** 微信小程序；Node.js；Express；MySQL；智能客服；电商系统

## Abstract

With the continuous development of mobile Internet and the WeChat ecosystem, mini programs have become an important carrier for small and medium-sized e-commerce systems due to their lightweight usage and relatively low development cost. To address the problems of fragmented functions, inconsistent user experience, and inefficient customer service in traditional shopping mini programs, this paper designs and implements an intelligent shopping mini program based on Node.js and JavaScript. The system adopts a front-end and back-end separated architecture. The front end is developed with uni-app for the WeChat mini program interface, the back end is built with Express for RESTful APIs, and MySQL is used for persistent data storage. An AI customer service module is also integrated through third-party large language model APIs.

The system focuses on the complete business workflow of product browsing, intelligent consultation, cart management, order placement, order tracking, and after-sales handling. It implements user authentication and profile management, product classification and search, product details browsing, shopping cart management, order lifecycle control, AI chat consultation, and backend administration. Compared with traditional e-commerce mini programs, the system introduces context-aware question answering on the product detail page, allowing users to ask about product attributes, usage instructions, logistics, and after-sales services in natural language, thereby improving interaction efficiency and user experience.

This paper also discusses front-end and back-end API design, database schema design, transaction processing, and stability control for AI API calls. Test results show that the system can satisfy the basic functional requirements of shopping scenarios and has good scalability and practical value.

**Key words:** WeChat Mini Program; Node.js; Express; MySQL; Intelligent Customer Service; E-commerce System

## 1 绪论

### 1.1 研究背景

近年来，移动端购物场景不断深化，用户对商品浏览、下单、售后和客服响应速度的要求持续提高。微信小程序由于无需安装、打开即用、适配微信生态等优势，已经广泛应用于校园商城、社区团购、个体店铺和轻量级电商服务中。但从实际应用来看，许多购物类小程序仍停留在“能展示、能下单”的基础阶段，存在商品信息承载不足、订单管理链路不完整、客服功能弱、页面交互割裂等问题。

与此同时，大模型技术的发展使得自然语言交互能力能够以较低成本嵌入业务系统。对于电商场景而言，AI 不仅可以承担商品咨询，还可以辅助售前答疑、售后引导和常见问题处理，这为提升小程序体验提供了新的实现路径。因此，结合微信小程序、Node.js、MySQL 与 AI 能力构建一个完整的购物系统，具有较强的现实意义。

### 1.2 研究目的与意义

本文的研究目标是设计并实现一个可运行、可扩展、可演示的智能购物小程序系统，使其能够覆盖用户从浏览商品到完成订单和售后咨询的核心流程。

其意义主要体现在以下几个方面：

1. 在工程实践上，验证微信小程序前端与 Node.js 后端协同开发的可行性。
2. 在数据管理上，探索 MySQL 在购物业务中的建模方式和事务处理方法。
3. 在交互体验上，引入 AI 客服能力，增强用户对商品和服务信息的获取效率。
4. 在系统设计上，形成一套适合轻量级电商项目的实现思路和模块划分方式。

### 1.3 国内外研究现状

目前，国内外关于电商系统、小程序应用和智能客服的研究较为丰富。传统电商系统大多围绕商品展示、订单管理、支付处理和用户评价展开，而微信小程序则进一步降低了用户使用门槛，使轻量化电商服务得以快速部署。随着自然语言处理技术的发展，越来越多系统开始尝试引入智能客服、商品推荐和对话式导购能力。

从现有研究来看，相关工作主要集中在以下几个方向：一是基于微信小程序的购物系统设计；二是基于 Node.js 的后端服务构建；三是电商系统中的数据库设计与事务控制；四是 AI 在客服和导购中的应用。本文在已有研究基础上，将这几类能力整合到一个完整的小程序系统中，强调“业务闭环 + 智能咨询”的一体化设计。

## 2 需求分析

### 2.1 系统目标

本系统面向普通用户和管理员两类角色。普通用户主要完成商品浏览、搜索、加购、下单、查看订单和咨询商品等操作；管理员主要负责商品管理、订单处理、反馈查看和基础数据维护。

系统需要满足以下目标：

- 支持登录与用户身份识别。
- 支持商品分类展示、搜索和详情浏览。
- 支持购物车和订单的完整业务流程。
- 支持 AI 智能客服咨询。
- 支持后台管理功能。
- 支持基础的数据安全和接口安全控制。

### 2.2 功能需求

#### 2.2.1 用户管理

系统应支持用户登录、退出登录、个人信息查看与编辑、收货地址管理等功能。用户登录后，其会话状态应能够在一定时间内保持有效，避免频繁重复登录。

#### 2.2.2 商品管理

系统应支持商品分类展示、商品列表浏览、关键词搜索、商品详情查看以及商品图片展示等功能。商品详情页应包含价格、库存、规格、服务标签和推荐商品等信息。

#### 2.2.3 购物车与订单

用户可以将商品加入购物车、修改商品数量、删除商品，并在确认订单页生成订单。订单应包含待付款、待发货、已发货、已完成、已取消等状态，用于描述商品交易的全流程。

#### 2.2.4 AI 智能客服

系统应在商品详情页和聊天页中提供 AI 咨询入口。用户可围绕商品参数、使用方法、物流、售后等内容进行提问，系统应结合当前商品上下文生成回答。

#### 2.2.5 后台管理

管理员应能够管理商品、查看订单、处理售后申请和查看反馈信息，以保证系统运营端的基本可维护性。

### 2.3 非功能需求

系统还应满足以下非功能要求：

- 易用性：界面简洁，用户能够快速上手。
- 响应性：主要页面加载和接口请求应保持较好响应速度。
- 安全性：敏感信息应存储在后端，接口应进行身份校验。
- 可扩展性：模块划分清晰，便于后续扩展支付、评价等功能。
- 兼容性：适配主流微信客户端和常见移动设备。

### 2.4 可行性分析

从技术角度看，uni-app、Express、MySQL 与前后端分离模式均较为成熟，适合实现轻量级购物系统。从业务角度看，系统功能围绕常见电商闭环展开，逻辑清晰，实施风险较低。从开发成本角度看，该方案能够在个人毕业设计周期内完成实现与验证。

## 3 系统总体设计

### 3.1 总体架构

系统采用前后端分离架构，整体可分为四层：

1. 前端展示层：基于 uni-app 实现微信小程序页面。
2. 后端服务层：基于 Node.js 与 Express 提供 RESTful API。
3. 数据存储层：基于 MySQL 保存用户、商品、订单、聊天记录等数据。
4. 智能能力层：通过后端调用第三方大模型接口实现 AI 咨询。

### 3.2 功能模块设计

系统主要由以下模块组成：

- 用户登录与个人中心模块
- 商品展示与搜索模块
- 商品详情与推荐模块
- 购物车模块
- 订单确认与历史订单模块
- 售后申请模块
- AI 聊天咨询模块
- 后台管理模块

各模块之间通过统一接口进行通信，页面层主要负责展示和交互，业务逻辑尽量下沉到服务层，以提升代码可维护性。

### 3.3 数据库设计

数据库设计围绕业务闭环展开，核心数据表包括用户表、商品表、商品图集表、购物车表、订单表、订单明细表、聊天会话表和聊天消息表等。

设计时遵循以下原则：

1. 业务主键尽量保持简洁可读。
2. 高频查询字段建立必要索引。
3. 订单、聊天等流水数据与主实体适当拆分。
4. 图集、规格等一对多关系采用子表或结构化字段表达。

### 3.4 接口设计

系统接口采用 RESTful 风格，按照业务模块划分。前端通过统一请求层访问后端接口，后端负责参数校验、身份验证、业务处理和统一响应。

主要接口包括：

- 登录与用户信息接口
- 商品列表与详情接口
- 购物车接口
- 订单接口
- 售后接口
- AI 聊天接口
- 管理后台接口

#### 3.5.1 用户登录流程

用户进入系统后可通过登录页面完成身份认证。前端获取登录凭证后提交给后端，后端完成会话验证并返回用户信息与令牌，前端据此维护登录状态。

## 4 系统实现

### 4.1 前端页面实现

前端基于 uni-app 开发，采用页面式结构组织各业务模块。首页展示商品分类、热销商品和 AI 入口；商品详情页负责承载商品基本信息、规格选择和购买操作；购物车页用于集中管理待购买商品；个人中心页用于展示用户资料和订单入口。

在页面实现中，系统尽量复用统一组件，例如标题栏、商品卡片、空状态、操作按钮和消息气泡等，以降低页面间差异并提升维护效率。

### 4.2 后端服务实现

后端采用 Express 搭建接口服务，并按模块进行拆分。每个模块一般包含路由、控制器、服务和数据访问层。路由层负责接收请求，控制器层负责参数整理，服务层负责业务逻辑，数据层负责数据库读写。

这种分层方式可以减少页面逻辑与数据库逻辑的耦合，使系统更容易测试和扩展。例如，在订单创建场景中，服务层会先校验库存，再生成订单主表和明细表，最后更新库存状态，整个过程通过事务保证原子性。

### 4.3 商品模块实现

商品模块是系统的核心入口，主要包括商品列表、商品分类、搜索和详情展示。商品列表页面支持按分类筛选与关键词搜索，详情页则展示商品价格、库存、规格、标签和推荐信息。为了增强购物决策支持，商品详情页中还集成了智能咨询入口，用户可以基于当前商品上下文直接发起提问。

### 4.4 购物车与订单模块实现

购物车模块用于承接用户的临时购买意图。用户可自由增减商品数量并进入确认订单页。订单模块负责生成订单编号、保存订单明细、记录订单状态并展示历史订单。对于订单状态，系统采用清晰的状态流转设计，避免前后端对状态含义理解不一致。

### 4.5 AI 聊天模块实现

AI 聊天模块是本系统的特色功能。用户在聊天页面输入问题后，系统将商品名称、描述、规格、服务标签等信息拼接为上下文，再调用后端封装的 AI 接口。后端会对请求进行超时控制、异常处理和结果清洗，以提高稳定性。

为了保证用户体验，系统在 AI 返回异常时会给出友好提示，避免页面出现空白或卡死现象。同时，聊天记录会被保存，便于用户后续查看历史会话。

### 4.6 后台管理模块实现

后台管理模块主要供管理员使用，功能包括商品管理、订单管理、售后处理和反馈查看。管理员可对商品进行新增、编辑、上下架和图片维护，也可对订单状态进行查看和更新。该模块强调权限控制，非管理员用户不能访问相关操作。

## 6 结论与展望

### 6.1 结论

本文完成了一套基于 Node.js 与 JavaScript 的购物小程序设计与实现。系统围绕用户购物主链路展开，结合 uni-app、Express、MySQL 和 AI 能力，构建了一个前后端分离、模块清晰、功能完整的智能购物系统。实践表明，该方案能够较好地满足轻量级电商应用的开发需求。

### 6.2 展望

后续可以从以下几个方向继续完善：

1. 接入真实支付能力，完成更完整的交易闭环。
2. 增加商品评价和推荐算法，提高个性化程度。
3. 优化后台统计看板，增强运营管理能力。
4. 扩展 AI 能力，使其支持更细粒度的导购和售后场景。
5. 引入更完善的日志与监控机制，提升系统稳定性。
